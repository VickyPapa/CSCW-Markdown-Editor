/*
 * 说明：
 * 
 * 1、使用：
 * 可以使用jar包运行或使用eclipse工程文件运行。
 * 请先启动服务器，再启动客户端。
 * 
 * 另注一个小BUG的说明：
 * 如果一运行程序就直接打开文件，会出现循环刷新的现象。
 * 如果是键入，或者先有键入再打开文件，不会循环刷新的。
 * 所以如果要测试打开文件的同步功能，麻烦在编辑域键入一下。
 * （有小天使能在评语里告诉我为什么会这样吗...
 * 我觉得打开文件和键入是一样的来着，代码里有对循环刷新做处理_(:зf∠)_）
 * 
 * 2、支持功能
 * 支持一个客户端、两个客户端、两个以上客户端这样三种模式。
 * 支持编辑域同步更新。
 * 支持预览域同步更新。
 * 支持导航栏域同步更新。
 * 支持导入md文件同步更新。
 * 支持导入自定义css文件同步更新。
 * （因为我理解的鼠标同步最后都是体现为导入文件或者编辑域的变化，
 * 如果真的鼠标一起动的话感觉就是同步桌面而不是CSCW了）
 * 
 * 3、自带的测试材料
 * test.md
 * test.css(会变绿变绿变绿)
 * 欢迎使用自己的测试材料
 *                     
 * 4、推荐的test cases
 * (1)键入“123”（测试文本域的同步更新、测试预览域的同步更新）
 * (2)键入“1”，打开tets.md（测试导航栏的同步更新、测试md文件的同步更新）
 * (3)在(2)之后导入自定义css，或者直接键入再导入css（测试自定义css文件同步更新）
 * (4)打开一个客户端键入（测试一个人的时候是否能够独自坚强工作...）
 * (5)打开两个客户端键入（测试队对两个客户端的支持）
 * (6)打开三个客户端键入（测试队对多个客户端的支持）
 * ......
 * 欢迎使用更多测例。
 */
